Quick tips

  • Start with issues labelled good first issue or documentation.
  • Read the project's CONTRIBUTING.md and CODE_OF_CONDUCT.
  • Make meaningful, well-described PRs — small & helpful changes are perfect.

How to participate

1. Register

Sign up at the Hacktoberfest website so your PRs are tracked and eligible for rewards.

2. Find projects

Search on GitHub/GitLab for labels like hacktoberfest, help wanted, or good first issue.

3. Make contributions

Open PRs for fixes, documentation, translations, or small features. Keep PRs focused and respectful.

Starter project ideas

Docs & Guides

Improve README, add code samples, or write contributor-friendly guides.

UI Fixes

Tiny stylesheet fixes, accessibility improvements, or responsive tweaks.

Bugfixes & Tests

Fix a failing test, refactor a small function, or add unit tests.

Dos & Don'ts

Do

  • Explain your changes clearly in the PR description.
  • Link related issues and tests.
  • Be patient and polite with maintainers.

Don't

  • Don't open spammy or trivial PRs that add no value.
  • Don't ignore contribution guidelines.
  • Don't claim credit for work you didn't do.

FAQ

Q: What counts as an accepted PR?
An accepted PR is one that has been merged by a project maintainer or explicitly accepted per the project's rules.

Q: Can documentation count?
Yes — docs, translations, and design contributions are valid if the project maintainer accepts them.

Q: Where can I get help?
Look for community channels, Discord/Slack links on the project repo, or ask maintainers politely on the issue/PR.
